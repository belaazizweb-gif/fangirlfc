Phase 7 — Saved Card Flow + Dashboard + Mobile Download Fix.

Read the relevant files first, then implement only the fixes below.

Do NOT modify:
- penalty gameplay
- Football IQ questions/content
- progression economy
- badge logic
- quiz scoring
- FanCard visual design except where needed for export/share buttons
- landing hero unless strictly necessary
- Firestore rules unless required for share link permissions

Current problem:
The app works, but the user flow is not clear enough after the first experience.

We need a clear distinction:

Guest Mode:
User can try the app freely:
Quiz → Card → Football IQ → Penalty → updated card.

Saved Mode:
User signs in to keep:
Fangirl Card, stars, badges, IQ level, penalty score, and share link.

Do NOT force sign-in before the quiz.
But after the user creates value, guide her clearly to sign in.

Use public wording:
- “Save your Fangirl Card”
- “Keep your stars and badges”
- “Continue later”
- “Create a shareable card link”

Do NOT use:
- “Official”
- “FIFA”

4. Separate Download and Share on the card page

Current issue:
“Share / Save card” is confusing. On mobile it opens the share sheet, but user may want direct image download.

Create separate actions:

Primary button:
“Download image”

Secondary button:
“Share card”

Third action if available:
“Copy share link”

Download image behavior:
1. Generate PNG from cardRef.
2. Wait for all images inside cardRef to fully load/decode before export.
3. Trigger direct download if browser allows.
4. If mobile blocks download, show a modal with the generated image and text:
“Long press the image to save.”

Share card behavior:
1. Generate PNG from cardRef.
2. Use Web Share API with file if supported.
3. If not supported, fallback to public share link / copied link.

Download must NOT open WhatsApp by default.
Share may open WhatsApp if user chooses it from native share sheet.

---

5. Fix blank or missing-photo export bug

Bug:
A user shared/downloaded the card and the generated image was blank or missing the uploaded photo.

Required fix:
Before exporting cardRef:
- wait for all images inside cardRef
- use img.decode() when available
- fallback to onload promise
- ensure uploaded photo is stored/rendered as a stable data URL
- show loading state: “Preparing your card…”

Add helper if needed:
- waitForImages(cardRef)
- exportCardImage(cardRef)

Acceptance:
- uploaded gallery photo appears in exported PNG
- selfie photo appears in exported PNG
- shared image is not blank
- WhatsApp-shared image includes the card photo
- outside UI is not included in export
