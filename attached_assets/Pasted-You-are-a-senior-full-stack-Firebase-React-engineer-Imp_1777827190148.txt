# GOAL

Move official progression authority from the client to Firebase Cloud Functions.

Currently, the client can directly update:

* `users/{uid}.officialStars`
* `users/{uid}.officialXp`
* `users/{uid}.rankScore`
* `users/{uid}.officialStarActions`
* `teams/{teamCode}.totalStars`
* `teams/{teamCode}.totalXp`
* `teams/{teamCode}.rankScore`
* `leaderboard/{uid}`

This is acceptable for MVP, but not secure.

After Phase 4:

* Client may still set official card/team.
* Client must NOT directly award official stars.
* Client must NOT directly update team totals.
* Cloud Function becomes the only trusted path for official progression.

# REQUIRED ARCHITECTURE

Create callable Cloud Function:

```ts
awardOfficialStarsSecure({ action: string })
```

The function must:

1. Require authenticated user.
2. Use `context.auth.uid`.
3. Validate action against fixed allowlist.
4. Read `users/{uid}`.
5. Enforce once/cap logic server-side.
6. Calculate deltas server-side.
7. Update:

   * `users/{uid}`
   * `teams/{officialTeamCode}` if valid
   * `leaderboard/{uid}`
8. Use Firestore transaction for user + team updates.
9. Never trust stars/xp values from client.
10. Return safe result:

```ts
{
  awarded: boolean,
  action: string,
  starsDelta: number,
  xpDelta: number,
  officialStars: number,
  officialXp: number,
  rankScore: number,
  reason?: string
}
```

---

# STAR ACTION ALLOWLIST

Use exact values:

```ts
publish_official_card: {
  stars: 0.5,
  xp: 50,
  once: true
}

replace_official_card: {
  stars: 0.25,
  xp: 25,
  maxCount: 3
}

share_official_card: {
  stars: 0.5,
  xp: 50,
  once: true
}

complete_profile: {
  stars: 0.5,
  xp: 50,
  once: true
}
```

Rules:

* Unknown action → reject.
* once action already used → return `{ awarded: false, reason: "already_awarded" }`.
* maxCount reached → return `{ awarded: false, reason: "max_count_reached" }`.
* missing user profile → return safe error, do not create fake profile silently.

# TEAM AGGREGATION

If `users/{uid}.officialTeamCode` exists and is valid:

* update `teams/{teamCode}`:

  * increment `totalStars` by `starsDelta`
  * increment `totalXp` by `xpDelta`
  * increment `rankScore` by `xpDelta + starsDelta * 100`
  * preserve existing `memberCount`
  * ensure `teamCode`, `teamName`, `flag`, `updatedAt`

If no valid official team:

* skip team update safely.

Important:

* Do NOT move historical stars.
* Do NOT recalculate all users.
* Do NOT modify memberCount in this function.
