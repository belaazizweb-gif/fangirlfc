CANVAS SYSTEM
Use fixed logical canvas dimensions:
TEMPLATE_W = 1086
TEMPLATE_H = 1448

The JSON config uses normalized coordinates from 0.0 to 1.0.

Create helpers:
nX(value) = value * TEMPLATE_W
nY(value) = value * TEMPLATE_H
nW(value) = value * TEMPLATE_W
nH(value) = value * TEMPLATE_H

All element positions must come from the real config.

Do not hardcode random positions for:
- photo
- rating
- position
- flag
- badge
- name
- stats

MOBILE PREVIEW SCALING
The internal render coordinate system must remain 1086 × 1448.

For mobile display:
- calculate previewScale based on available width
- visually scale the Konva Stage
- keep all internal coordinates based on 1086 × 1448

Example principle:
const previewScale = availableWidth / TEMPLATE_W;

Stage:
width = TEMPLATE_W * previewScale
height = TEMPLATE_H * previewScale
scaleX = previewScale
scaleY = previewScale

Do not render the actual scene at 300 × 400.
Small dimensions are only visual display scaling, not the real render coordinate system.

FLAG RENDERING RULE FOR PHASE 1
Current flags are SVG files in public/flags/.

SVG in canvas/Konva can be inconsistent across browsers.
For Phase 1, do NOT block the render engine because of flags.

Implement one of these safe approaches:

Preferred Phase 1 approach:
- Try to load the SVG flag with use-image.
- If it renders correctly, show it.
- If it fails or causes issues, show a simple fallback:
  - colored rectangle placeholder
  - country code text, e.g. "FR"

Do NOT spend time converting all SVG flags to PNG in Phase 1.
Do NOT let SVG flag rendering break typecheck/build/export.
Do NOT use external remote flag URLs.
