TASK: Fangirl FC Phase 7 — Firestore Rules Hardening for Play Store Testing

CONTEXT
Fangirl FC is moving toward Play Store closed testing.

Current stable state:
- full app works
- Creator works
- MODNet background remover works
- Auth redirect works
- TWA assetlinks deployed
- Android app added in Firebase
- AAB/APK generated

Known risk from audit:
Firestore leaderboard rules may allow authenticated users to write arbitrary progression fields to leaderboard/{uid}, including officialStars, officialXp, rankScore.
Progression/official stars should be server-controlled, not client-controlled.

Goal:
Audit and harden Firestore rules for Play Store testing without breaking normal app flows.

- do not expose secrets

FILES ALLOWED:

2. firebase.json only if rules path is missing or incorrect
3. Cloud Functions files read-only unless a critical mismatch is found

==================================================
PHASE 1 — HARDEN LEADERBOARD WRITES
==================================================

Goal:
Users must not be able to directly write server-controlled progression fields.

Server-controlled fields include:
- officialStars
- officialXp
- rankScore
- officialRank
- rank
- totalStars
- totalXp
- updatedByFunction
- any field used to determine leaderboard order

Rules should allow authenticated user to create/update only safe profile/display fields if needed, such as:
- displayName
- photoURL
- team
- identityId
- templateId
- updatedAt

But block direct client writes to:
- officialStars
- officialXp
- rankScore
- rank
- totalStars
- totalXp
- any progression/ranking score

If current client does not need to write leaderboard directly, then make leaderboard client writes false and only allow reads.

Prefer the safest rule that does not break app flows.

==================================================
PHASE 2 — HARDEN USERS AND TEAMS IF NEEDED
==================================================

Users:
- User can read/write own basic profile.
- User must not directly write official progression fields if Cloud Function controls them.

Teams:
- Public read can remain.
- Direct client writes to team score/rank fields must be blocked.
- Cloud Function/admin/server should be only writer for aggregate ranking fields.

Do not block normal account/profile creation if existing app needs it.
