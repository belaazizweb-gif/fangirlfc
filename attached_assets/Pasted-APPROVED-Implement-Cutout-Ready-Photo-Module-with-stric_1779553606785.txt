PHASE 4 — TRANSPARENT PNG DETECTION
==================================================

In CreatorScreen.tsx, add:

async function detectMeaningfulTransparency(dataUrl: string): Promise<boolean>

Only run it when:
file.type === "image/png"
OR
file.name.toLowerCase().endsWith(".png")

Do not run alpha detection for JPG/WebP.

Detection rules:
- load the uploaded dataUrl into an Image
- wait for image load/decode safely
- draw to temporary canvas
- downsample so longest side <= 512px
- inspect alpha channel
- count pixels where alpha < 250
- transparentRatio = transparentPixelCount / totalPixelCount
- return transparentRatio > 0.01

Failure behavior:
- if image load fails → return false
- if canvas fails → return false
- if dimensions are invalid → return false
- if detection throws → return false

Important:
- PNG with baked checkerboard but no alpha must return false
- opaque PNG must return false
- transparent PNG cutout must return true
- upload flow must never crash because of detection

No external library.

PHASE 5 — HANDLE PHOTO UPLOAD
==================================================

In handlePhotoSelect, preserve the existing flow and x/y convention.

Before setting photo state:
- read file as dataUrl
- get naturalWidth/naturalHeight from the image
- detect isTransparentPng only if file is PNG
- branch by result

Normal photo branch:
- src = dataUrl
- cutoutSrc = null
- isCutout = false
- use getPhotoBox
- scale formula:

Math.max(photoBoxWidth / naturalWidth, photoBoxHeight / naturalHeight) * 1.02

- x = photoBox center X
- y = photoBox center Y
- rotation = 0

Transparent PNG cutout branch:
- src = dataUrl
- cutoutSrc = dataUrl
- isCutout = true
- use getCutoutBox
- scale formula:

Math.min(cutoutBoxWidth / naturalWidth, cutoutBoxHeight / naturalHeight) * 0.98

Important:
Math.min is intentional for cutout mode.
Do not “correct” it to Math.max.
Reason:
- cutout must fit fully inside the player zone
- no part of the transparent player should be cropped by initial fit
- normal photo uses Math.max because it must cover a rectangular photoBox
- cutout uses Math.min because it must remain fully visible as a player layer

- x = cutoutBox center X
- y = cutoutBox center Y
- rotation = 0

Critical reset rule:
When uploading a normal photo after a cutout, cutoutSrc must become null and isCutout must become false.
Do not leave stale cutoutSrc active.
