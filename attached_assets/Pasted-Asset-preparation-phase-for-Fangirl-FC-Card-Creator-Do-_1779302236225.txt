Asset preparation phase for Fangirl FC Card Creator.

Do NOT build the Player Card Creator module yet.
Do NOT modify existing app UX, routes, dashboard, quiz, Learn, Penalty, auth, or export logic.
This task is only to prepare reusable assets for the upcoming Card Creator.

Context:
We are preparing the new Player Card Creator module.
Before coding the module, we need:
1. D-DIN fonts installed locally in the project
2. All country flags copied locally into public/flags
3. Basic card creator asset structure prepared
4. Basic countries/positions config files prepared

Important:
I will upload a file named d-din.zip into the Replit project root before you start.
Use shell commands where appropriate.

Tasks:

1. Extract D-DIN font zip

There will be a file in project root:

d-din.zip

Extract it and copy these fonts into:

public/fonts/d-din/

Required font files:
- D-DIN.otf
- D-DIN-Bold.otf
- D-DINCondensed.otf
- D-DINCondensed-Bold.otf

Keep the license file if present:
- SIL Open Font License.txt

If the zip contains additional D-DIN variants, keep them only if easy, but the four files above are mandatory.

2. Register D-DIN fonts in global CSS

Add @font-face definitions for:

- D-DIN
- D-DIN Bold
- D-DIN Condensed
- D-DIN Condensed Bold

Use local paths:

/fonts/d-din/D-DIN.otf
/fonts/d-din/D-DIN-Bold.otf
/fonts/d-din/D-DINCondensed.otf
/fonts/d-din/D-DINCondensed-Bold.otf

Do not apply the font globally to the whole app yet.
Only register the fonts so the future Card Creator can use them.

3. Install and copy all country flags

Use the GitHub package/source:

flag-icons by lipis

Preferred method:

npm install flag-icons
mkdir -p public/flags
cp -r node_modules/flag-icons/flags/4x3/* public/flags/

After copying, the app should have files like:

public/flags/fr.svg
public/flags/us.svg
public/flags/ma.svg
public/flags/br.svg
public/flags/ar.svg
public/flags/es.svg
public/flags/de.svg
public/flags/pt.svg
public/flags/gb-eng.svg

If the package is only used to copy assets, you may leave it installed or uninstall it after copying. Prefer the safest option for the current project.
