==================================================
PHASE 2 — ADD public/sw.js
==================================================

Create public/sw.js.

Strategy must be conservative and safe.

Requirements:

1. Install event:
- cache only minimal app shell assets:
  - /
  - /offline.html
  - /manifest.json
  - /icon.svg
  - /icons/icon-192.png
  - /icons/icon-512.png
  - /icons/icon-maskable-192.png
  - /icons/icon-maskable-512.png

2. Activate event:
- clean old caches by cache name/version
- call self.clients.claim()

3. Fetch event:
- for navigation requests:
  - try network first
  - if network fails, return /offline.html from cache

- for same-origin static assets:
  - use cache-first only for:
    /_next/static/
    /icons/
    /icon.svg
    /manifest.json

- do NOT cache:
  - Firebase/Firestore/Auth requests
  - Google OAuth/Auth URLs
  - Hugging Face model/CDN requests
  - uploaded images/blob/data URLs
  - dynamic share URLs aggressively
  - API calls
  - POST requests

4. Do not intercept non-GET requests.

5. Do not throw if caching fails.

Cache name:
fangirl-fc-v1

PHASE 3 — REGISTER SERVICE WORKER
==================================================

Add a tiny client component:

components/ServiceWorkerRegister.tsx

Behavior:
- register /sw.js only in production
- only if "serviceWorker" in navigator
- wait until window load
- catch and console.warn registration failures
- no UI

Example logic:
if (process.env.NODE_ENV === "production" && "serviceWorker" in navigator) {
  window.addEventListener("load", () => {
    navigator.serviceWorker.register("/sw.js").catch(...)
  })
}

Mount this component once in the root layout.

Do not register multiple times.
Do not use next-pwa unless necessary.
