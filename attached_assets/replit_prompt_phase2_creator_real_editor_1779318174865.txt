PHOTO DRAG TECHNICAL RULE
The card state must store photo.x and photo.y in logical 1086 × 1448 coordinates, not screen pixels.

If using pointer/client coordinates or drag deltas from the scaled preview:
- convert screen pixels back to logical coordinates by dividing by previewScale before updating cardState.photo.x and cardState.photo.y

If using Konva node.x() / node.y() and the values are already in Stage logical coordinates:
- do NOT divide again

The final saved cardState.photo.x and photo.y must always be logical offsets relative to the 1086 × 1448 canvas.

This avoids photo jumping or incorrect movement on mobile.

PHOTO RENDERING RULE
The photo must be:
- clipped to the layout.photo rectangle
- rendered before the masked overlay
- positioned using logical coordinates
- scaled using cardState.photo.scale
- rotated using cardState.photo.rotation
- exported exactly as previewed

Use the existing masked overlay engine from Phase 1.5.

Do not break the current overlay mask.
