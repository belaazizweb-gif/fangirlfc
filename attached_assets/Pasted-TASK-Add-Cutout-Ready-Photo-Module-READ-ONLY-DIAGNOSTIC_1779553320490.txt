PHASE 5 — DEFAULT FIT LOGIC

Normal photo initial fit:
- keep current behavior:
  Math.max(photoBoxWidth / imageWidth, photoBoxHeight / imageHeight) * 1.02
- use getPhotoBox
- center inside photoBox

Transparent cutout initial fit:
- use getCutoutBox
- preserve image ratio
- fit based on cutoutBox height/width using Math.min, not Math.max

Use:

const scale =
  Math.min(cutoutBoxWidth / imageWidth, cutoutBoxHeight / imageHeight) * 0.98;

Reason:
- Math.min is intentional for cutout mode
- cutout must fit inside the player zone without being cropped
- normal photo uses Math.max because it must cover its rectangular photoBox
- cutout uses Math.min because it must remain fully visible as a player layer
- do not “correct” cutout fitting to Math.max

Then center inside cutoutBox:
- x = cutoutBox center X
- y = cutoutBox center Y
- rotation = 0

If this makes the cutout too small during screenshot validation, adjust only multiplier between 0.98 and 1.08.
