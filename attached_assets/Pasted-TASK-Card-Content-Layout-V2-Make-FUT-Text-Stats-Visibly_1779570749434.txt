- rating/position/name do not feel premium enough

- overall content composition still looks too close to the old version

Make the card content visibly more professional without touching image/cutout/export systems.

PHASE 2 — CHANGE STATS RENDERING TO INLINE FUT ROWS

Current stat rendering is value above label.
Replace it with inline FUT-style rows.

Instead of:

99
PAC

Render:

99 PAC

Layout:
- keep two-column, three-row structure
- left column: PAC, SHO, PAS
- right column: DRI, DEF, PHY
- each row uses the existing stat row zone
- value and label are on the same baseline

For each stat row:
- value takes about 42% of the row width
- label takes about 58% of the row width
- value is right-aligned
- label is left-aligned
- small horizontal gap between value and label

Pseudo layout:

const valueW = sw * 0.42;
const labelW = sw * 0.52;
const gap = sw * 0.04;

value:
x = sx
width = valueW
align = "right"

label:
x = sx + valueW + gap
width = labelW
align = "left"

Vertical:
- both value and label centered vertically inside the row
- do not split row into 55% / 40% boxes anymore

Font sizes:
- stat value fontSize = sh * 0.58
- stat label fontSize = sh * 0.46
- label opacity = contentProfile.statLabelOpacity
- value opacity = 1

Keep:
- D-DIN Condensed
- bold
- profile shadow logic

This change must be visually obvious.

PHASE 3 — STATS DIVIDER

Keep the stats divider.

- divider renders behind stat text
- divider is subtle
- divider does not cover text
- divider is at least 1.5px wide after pixel conversion

PHASE 4 — RATING / POSITION IMPROVEMENT

Make rating and position more premium.

Rating:
- fontSize = ratingH * max(contentProfile.ratingFontRatio, 0.84)
- verticalAlign = "middle"
- align = "center"
- keep profile shadow
- no clipping

Position:
- fontSize = posH * max(contentProfile.positionFontRatio, 0.78)
- verticalAlign = "middle"
- align = "center"
- keep profile shadow

Goal:
- rating must dominate the left column
- position must be readable on mobile

PHASE 5 — NAME IMPROVEMENT

Goal:
