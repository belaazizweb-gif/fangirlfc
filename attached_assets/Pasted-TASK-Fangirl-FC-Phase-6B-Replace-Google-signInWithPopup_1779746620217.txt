==================================================
PHASE 1 — REPLACE GOOGLE POPUP WITH REDIRECT
==================================================

In the Google login helper:

Replace:
signInWithPopup(auth, provider)

With:
signInWithRedirect(auth, provider)

The Google login function should:
- start redirect flow
- not expect a UserCredential return immediately
- keep error handling for redirect initiation failures

Example pattern:

export async function signInWithGoogle() {
  const auth = getFirebaseAuth();
  if (!auth) throw new Error("Firebase Auth is not configured");

  const provider = new GoogleAuthProvider();
  provider.setCustomParameters({ prompt: "select_account" });

  await signInWithRedirect(auth, provider);
}

Do not change email/password functions unless necessary.

==================================================
PHASE 2 — HANDLE REDIRECT RESULT ON APP LOAD
==================================================

In AuthProvider/AuthContext:

Add getRedirectResult handling once on client load.

Behavior:
- after Firebase Auth is ready, call getRedirectResult(auth)
- if a UserCredential is returned, sync/create the user profile exactly like the popup flow used to do
- if result is null, do nothing
- catch and store/log redirect errors without breaking the app
- do not create duplicate users if onAuthStateChanged already handles profile sync

Important:
Avoid double profile writes if both getRedirectResult and onAuthStateChanged run.
Preferred safe approach:
- keep onAuthStateChanged as source of truth for current user state
- use getRedirectResult only to capture redirect errors and optionally trigger existing sync function if needed
- if existing onAuthStateChanged already syncs profile, do not duplicate sync logic
